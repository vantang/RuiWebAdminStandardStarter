# 测试与验收文档目录

业务项目应将测试计划、执行结果、缺陷复测和验收结论统一放在 `docs/qa/`。
对于真实业务项目，`docs/qa/*` 属于最少必备文档的一部分，不建议省略。

推荐内容：

- 首版测试与验收方案
- 版本回归结果
- 缺陷复测记录
- 验收结论与未覆盖项

推荐起手模板：

- `docs/templates/业务项目测试验收模板.md`

目录中的文档应与以下内容保持一致：

- `README.md`
- `docs/tech/*`
- `docs/api/*`

缺陷分级统一使用 `docs/standards/05-文档规范与交付清单.md` 中定义的 `P0 / P1 / P2 / P3`。
