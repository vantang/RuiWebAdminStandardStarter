# 后台模板技术方案

## 1. 总体结构

本模板采用三段式结构：

```text
Vue 前端 DEMO
      |
      | HTTP / JSON
      v
独立 Mock Server
      |
      v
规范与技术文档
```

## 2. 分层目的

- `web`：承载页面布局、组件模式和交互演示
- `mock-server`：模拟后端接口，支撑前后端分离开发
- `docs`：沉淀规范和项目可复用资产

## 3. 前端设计

前端保留以下长期稳定层：

- `layout`：后台主框架
- `router`：统一页面组织方式
- `styles/base.css`：颜色、圆角、阴影、字体等设计 token
- `styles/admin.css`：页头、卡片、表格、布局等页面模式

业务可替换层：

- `views/examples`
- `api/modules.js`
- 业务专属组件

## 4. Mock 设计

Mock 服务采用 Node.js 原生 HTTP，目的不是追求复杂度，而是：

- 启动轻量
- 易于阅读
- 便于复制到新项目
- 适合在受限环境下演示

## 5. 推荐演进方向

当模板长期稳定后，可以继续补充：

- TypeScript 化
- 规范工具链
- 权限壳
- 用户信息和组织切换
- 暗色主题或主题切换
