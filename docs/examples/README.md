# 业务项目启动示例包

这个目录提供一套“填好的参考答案”，用于帮助你理解业务项目首次启动时的文档完成形态。

和 `docs/templates/` 的区别：

- `docs/templates/`：空白模板，适合直接复制后按项目填写
- `docs/examples/`：完整示例，适合理解最终应该写成什么样

## 当前示例主题

当前提供一套“以年终奖管理为例”的业务项目启动示例包：

1. [技术方案示例](./年终奖管理/技术方案示例.md)
2. [接口文档示例](./年终奖管理/接口文档示例.md)
3. [测试验收示例](./年终奖管理/测试验收示例.md)

## 使用建议

建议按以下顺序使用：

1. 先阅读 `docs/templates/业务项目初始化清单.md`
2. 再对照本目录示例，理解一份“合格的业务化文档”应该覆盖哪些信息
3. 最后回到 `docs/templates/*`，按你的实际业务填写

## 注意事项

- 这里的示例内容只用于演示文档形态，不代表模板仓库推荐某个固定业务方向
- 示例中的业务名词、接口路径、字段和流程不能直接当作你的业务事实
- 复制到真实业务项目后，必须按实际需求替换，不得保留为模板残留
