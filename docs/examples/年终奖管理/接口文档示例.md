# 年终奖管理接口文档示例

## 1. 文档说明

本文档用于说明“年终奖管理模块”首期建设中的核心接口定义，服务于方案配置、奖金测算、审批确认和导出查询。

适用范围：

- 前端页面开发
- Mock 联调
- 后端接口对齐
- 验收与回归检查

统一约定：

- Base Path：`/api`
- 响应结构：`{ code, message, data }`
- 鉴权方式：`Bearer Token`
- 当前版本：`v1`

## 3. 通用约定

### 3.1 成功响应

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

### 3.2 分页约定

列表接口默认约定：

- 请求参数：`page`、`pageSize`
- 响应字段：`list`、`page`、`pageSize`、`total`

### 3.3 状态字段约定

| 字段 | 可能值 | 说明 | 页面表现 |
| --- | --- | --- | --- |
| `bonusStatus` | `pending_confirm / confirmed / pending_fix / exported` | 奖金处理状态 | 列表 Tag、按钮显隐 |
| `approvalResult` | `pending / approved / rejected` | 审批结果 | 详情轨迹、审批操作区 |
| `exceptionFlag` | `true / false` | 是否存在异常 | 列表警示、详情提示块 |

## 4. 资源接口明细

### 4.1 奖金测算列表

- Method：`GET`
- Path：`/api/bonus/calculations`
- 业务说明：查询员工年终奖测算结果，供 HR、负责人和财务查看与筛选
- 调用页面：`/bonus/calculations`

请求参数：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `keyword` | `string` | 否 | 员工姓名 / 工号模糊搜索 |
| `departmentId` | `string` | 否 | 按部门筛选 |
| `bonusStatus` | `string` | 否 | 按奖金状态筛选 |
| `page` | `number` | 否 | 页码，默认 1 |
| `pageSize` | `number` | 否 | 每页条数，默认 10 |

响应示例：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "list": [
      {
        "id": "B2025-001",
        "employeeName": "张敏",
        "departmentName": "华东销售部",
        "bonusAmount": 32000,
        "bonusStatus": "pending_confirm",
        "exceptionFlag": false
      }
    ],
    "page": 1,
    "pageSize": 10,
    "total": 1
  }
}
```

字段说明：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `employeeName` | `string` | 员工姓名 |
| `bonusAmount` | `number` | 测算后的奖金金额 |
| `bonusStatus` | `string` | 当前奖金状态 |
| `exceptionFlag` | `boolean` | 是否命中异常规则 |

错误码：

| code | message | 说明 |
| --- | --- | --- |
| `4001` | `invalid_query` | 查询参数非法 |

Mock / 联调说明：

- Mock 数据位置：`mock-server/src/data/bonus.js`
- 前端调用位置：`web/src/api/modules.js`
- 注意事项：`bonusStatus` 由后端最终返回，前端不自行组合状态

### 4.2 审批确认动作

- Method：`POST`
- Path：`/api/bonus/approvals/:id/action`
- 业务说明：部门负责人对待确认奖金记录执行确认或驳回
- 调用页面：`/bonus/approvals`

请求参数：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `action` | `string` | 是 | `approve` 或 `reject` |
| `reason` | `string` | 否 | 驳回原因，驳回时必填 |

响应示例：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": "B2025-001",
    "approvalResult": "approved"
  }
}
```

错误码：

| code | message | 说明 |
| --- | --- | --- |
| `4002` | `reason_required` | 驳回时未填写原因 |
| `4003` | `invalid_status` | 当前状态不允许审批 |

Mock / 联调说明：

- Mock 数据位置：`mock-server/src/data/bonus.js`
- 前端调用位置：`web/src/api/modules.js`
- 注意事项：驳回时必须透传 `reason`，否则由接口直接返回 `reason_required`

### 4.3 导出任务列表

- Method：`GET`
- Path：`/api/bonus/exports`
- 业务说明：查询奖金导出任务状态，供 HR 与财务查看导出进度和下载入口
- 调用页面：`/bonus/exports`

请求参数：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `batchNo` | `string` | 否 | 按奖金批次筛选 |
| `status` | `string` | 否 | 按导出状态筛选 |
| `page` | `number` | 否 | 页码，默认 1 |
| `pageSize` | `number` | 否 | 每页条数，默认 10 |

响应示例：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "list": [
      {
        "id": "EXP-2025-001",
        "batchNo": "BONUS-2025-Q4",
        "status": "completed",
        "fileName": "bonus-export-2025-q4.xlsx",
        "createdAt": "2026-01-08 10:30:00"
      }
    ],
    "page": 1,
    "pageSize": 10,
    "total": 1
  }
}
```

字段说明：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `batchNo` | `string` | 导出批次号 |
| `status` | `string` | 导出状态，取值如 `queued / processing / completed / failed` |
| `fileName` | `string` | 导出文件名 |
| `createdAt` | `string` | 导出任务创建时间 |

错误码：

| code | message | 说明 |
| --- | --- | --- |
| `4004` | `invalid_export_status` | 导出状态参数非法 |

Mock / 联调说明：

- Mock 数据位置：`mock-server/src/data/bonus.js`
- 前端调用位置：`web/src/api/modules.js`
- 注意事项：下载链接可由后端后续补充，当前首版只验证任务查询与状态展示

## 5. Mock 与联调策略

- 当前前端默认调用：本地 `mock-server`
- Mock 覆盖范围：奖励方案、测算列表、详情查询、审批动作、导出任务
- 联调切换方式：Vite 代理切换到测试环境网关
- 未提供接口的临时方案：继续以本地 Mock 作为首期交付基线，后续再切换测试环境网关
