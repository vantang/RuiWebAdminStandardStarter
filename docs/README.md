# 文档总览

这个目录用于承载后台模板的规范、技术方案、接口示例、测试验收文档、示例包和文档模板。

## 目录说明

- `standards/`：技术、设计、代码、文档规范
- `tech/`：模板工程的实现方案和结构说明
- `api/`：通用后台 DEMO 的接口说明
- `qa/`：业务项目最少必备的测试与验收文档
- `examples/`：业务项目启动示例包
- `templates/`：可直接复制的文档模板、AI 提示词模板与检查清单

## 建议阅读顺序

1. `standards/01-项目定位与复用方式.md`
2. `standards/02-技术栈与目录规范.md`
3. `standards/03-设计系统与页面模式.md`
4. `standards/04-代码规范与协作约定.md`
5. `standards/05-文档规范与交付清单.md`
6. `standards/06-提交与分支规范.md`
7. `standards/07-AI协作与业务化替换规范.md`
8. `tech/后台模板技术方案.md`
9. `api/通用后台Demo接口.md`

## 使用分流

如果你正在维护本模板仓库本身，优先阅读 `standards/` 与 `tech/`。

如果你正在把本仓库复制成真实业务项目，优先阅读：

1. `templates/业务项目初始化清单.md`
2. `examples/README.md`
3. `templates/业务项目README模板.md`
4. `templates/业务项目技术方案模板.md`
5. `templates/业务项目接口文档模板.md`
6. `templates/业务项目测试验收模板.md`
7. `templates/AI业务项目启动提示词模板.md`
8. `templates/模板残留检查清单.md`
9. `standards/07-AI协作与业务化替换规范.md`

## 模板与示例怎么选

如果你已经知道要写什么，但还缺一份可直接复制的骨架，优先使用 `templates/`。

如果你不确定一份“合格的业务化文档”最终应该写成什么样，先阅读 `examples/`，再回到 `templates/` 按你的实际业务填写。

建议组合方式：

1. 先看 `templates/业务项目初始化清单.md`
2. 再看 `examples/README.md`
3. 最后按 `templates/业务项目README模板.md`、`业务项目技术方案模板.md`、`业务项目接口文档模板.md`、`业务项目测试验收模板.md` 逐个落稿
