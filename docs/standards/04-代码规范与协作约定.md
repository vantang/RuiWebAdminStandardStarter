# 代码规范与协作约定

## 1. 基本原则

- 优先清晰，再谈技巧。
- 页面逻辑和展示逻辑分层。
- 组件命名必须表达用途，而不是历史业务语义。
- 样式和结构尽量复用，不在页面内重复造轮子。

## 2. Vue 代码约定

- 使用 `script setup`
- 组合式 API 优先
- 页面数据请求放在页面层，纯展示组件尽量只接收 props
- 一个文件里避免同时承担布局、请求、复杂计算三类职责

## 3. API 约定

- `http.js` 只负责基础请求封装
- `modules.js` 按资源拆分接口
- 页面不直接拼完整 URL
- 页面不直接处理重复的响应结构拆包逻辑

## 4. 组件约定

- 通用组件放 `components/`
- 组件只暴露必要 props 和事件
- 业务高度专属的组件不要沉淀进模板主干

## 5. 样式约定

- 全局变量放 `styles/base.css`
- 页面模式样式放 `styles/admin.css`
- 组件私有样式优先使用 `scoped`
- 避免把大量内联样式留在模板主干

## 6. 协作约定

- 新增页面前先判断是否属于已有页面模式
- 新增组件前先判断是否能在现有组件上抽象
- 修改模板主干时，要同步更新 `docs/standards`
- 历史业务案例不得直接回灌到模板主干
