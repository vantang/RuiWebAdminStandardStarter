# 文档规范与交付清单

## 1. 文档目录规范

- `docs/standards`：模板级规范
- `docs/tech`：实现方案和架构说明
- `docs/api`：接口文档
- `docs/templates`：模板文档

## 2. 一个完整后台项目最少应具备的文档

- 项目说明 README
- 页面/模块说明
- API 设计文档
- 技术方案文档
- 交付清单

## 3. 文档编写原则

- 先说明目标，再说明结构，最后说明细节
- 路径、模块名、接口名保持和代码一致
- 不用大段空话替代具体目录和约束
- 面向接手的人写，而不是面向作者自己写

## 4. 建议交付清单

- 可运行前端
- Mock 服务或联调方式
- 目录结构说明
- 接口说明
- 页面截图或页面清单
- 风险说明和后续建议

## 5. 模板演进要求

- 每次模板主干发生明显变化时，至少更新一份规范文档
- 每次新增页面模式时，补充到设计规范
- 每次目录结构调整时，补充到技术规范
