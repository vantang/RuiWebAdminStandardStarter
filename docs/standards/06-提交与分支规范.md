# 提交与分支规范

## 1. 目标

这套模板的提交规范不是为了“好看”，而是为了：

- 让历史记录可读
- 让评审能快速定位改动范围
- 让模板主干和具体业务变更分离

## 2. 分支命名

默认建议：

- `main`：稳定主干
- `codex/<topic>`：由 Codex 推进的任务分支
- `feat/<topic>`：功能开发
- `fix/<topic>`：问题修复
- `docs/<topic>`：文档更新

## 3. 提交信息格式

建议格式：

```text
<type>(<scope>): <summary>
```

示例：

```text
feat(auth): add login shell and route guards
fix(layout): replace sidebar toggle icon
docs(standards): add commit and branch conventions
```

## 4. 推荐 type

- `feat`：新增功能
- `fix`：修复问题
- `refactor`：重构但不改变外部行为
- `docs`：文档更新
- `style`：纯样式或格式调整
- `chore`：脚手架、配置、依赖维护
- `test`：测试相关

## 5. 提交粒度要求

- 一个提交只解决一个明确问题
- 不要把文档、布局调整和业务逻辑改动混成一团
- 如果是模板主干改动，提交说明里要体现影响范围

## 6. 模板主干额外要求

- 修改 `layout`、`router`、`styles`、`docs/standards` 时，提交说明应明确写出影响的是“模板主干”
- 提交前先执行 `npm run check:strict`
