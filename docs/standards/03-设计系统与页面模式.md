# 设计系统与页面模式

## 1. 视觉目标

后台模板应保持：

- 专业、克制、稳定
- 信息层次清晰
- 高密度信息下仍易读
- 适合桌面端优先，同时兼顾中小屏

## 2. 基础视觉原则

- 使用统一颜色变量，不在页面内散落硬编码颜色。
- 圆角、边框、阴影统一来源于全局 token。
- 首页、列表页、详情页、文档页共用相同的页头结构。
- 卡片之间使用一致的间距体系。

## 3. 页面级模式

### 3.1 工作台

必须包含：

- 页头说明
- 指标卡
- 最近活动或任务
- 状态或进度区域

### 3.2 列表页

建议固定结构：

1. 页头
2. 筛选区
3. 操作区
4. 表格区
5. 分页区
6. 详情抽屉或二级操作

### 3.3 表单页

建议固定结构：

1. 页头说明
2. 按主题拆分的表单卡片
3. 底部操作栏
4. 成功反馈和校验反馈

### 3.4 详情页

建议固定结构：

1. 核心摘要
2. 基础信息
3. 进度/成员/记录
4. 时间线或日志
5. 附件与补充说明

## 4. 控件使用原则

- 筛选项优先使用 `Input`、`Select`、`DatePicker`
- 状态优先使用 `Tag`
- 强提醒优先使用 `Alert`
- 次级信息优先使用 `Descriptions`
- 大量结构化数据优先使用 `Table`
- 详情补充优先使用 `Drawer`
- 二次确认优先使用 `Dialog`

## 5. 响应式原则

- `>= 1200px` 以桌面布局为主
- `768px - 1199px` 控制列数和卡片换行
- `< 768px` 页头、筛选区、卡片布局自动纵向堆叠

## 6. 不建议的做法

- 页面内自行定义不同风格的标题区
- 一个页面里混用过多强调色
- 在后台页面中使用过重的装饰动效
- 组件命名和页面命名保留旧业务词汇
