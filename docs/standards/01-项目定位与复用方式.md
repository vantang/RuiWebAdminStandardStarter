# 项目定位与复用方式

## 1. 仓库定位

本仓库不是某个单独业务系统，而是一个“后台管理标准底座”。

它负责提供：

- 可直接运行的后台 DEMO
- 一套稳定的页面骨架和视觉语言
- 一套统一的接口组织方式
- 一套可以复制到新项目里的规范文档

## 2. 适用场景

适合以下项目作为起点：

- 管理后台
- 运营后台
- 审批后台
- 配置中心
- 数据看板型中后台

不建议直接用于：

- C 端营销页面
- 强品牌定制官网
- 复杂编辑器类产品

## 3. 复用策略

### 3.1 直接复制项目

适用于：

- 新建一个结构相近的后台项目
- 希望保留当前布局、组件风格和开发方式

步骤：

1. 复制仓库并重命名。
2. 替换 README、项目名称、菜单名称。
3. 保留 `layout`、`styles`、`router`、`api` 基础结构。
4. 用真实业务页面替换 `views/examples`。

### 3.2 只复用规范

适用于：

- 团队已有现成技术栈
- 只想借用设计规范和文档标准

做法：

- 复制 `docs/standards`
- 复制 `docs/templates`
- 参考 `web/src/styles` 中的变量和页面模式

### 3.3 只复用前端 DEMO

适用于：

- 需要快速演示后台布局和控件效果
- 需要和业务方先确认页面结构

做法：

- 保留 `web`
- Mock 数据按新业务换皮
- 文档页更新为当前项目自己的说明

## 4. 模板主干应长期保持稳定的部分

- 技术栈
- 目录分层
- 布局骨架
- 设计 token
- 页面模式
- 文档目录结构

## 5. 应按项目替换的部分

- 菜单结构
- 业务页面
- API 模块
- Mock 数据
- 业务名词
- 项目级 PRD 与交付文档
